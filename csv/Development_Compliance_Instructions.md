# Development Compliance Instructions

## 1. Issue Resolution

- If you encounter any issue or error, you are required to apply the correct and permanent fix.
- Shortcuts, temporary patches, or workaround solutions are **strictly prohibited**.
- Always identify and resolve the **root cause** before proceeding.
- All fixes must comply with **TypeScript strict mode** requirements.
- **Linting issues** must be resolved immediately; do not ignore or bypass them.

## 2. Feature Implementation

- When adding or implementing new features, strictly follow **established best practices and coding standards**.
- Do not use shortcuts, patches, or workaround implementations that may compromise **code quality**, **maintainability**, or **system integrity**.
- Ensure that all new code passes **TypeScript strict checks** and conforms to the project’s **ESLint** and **Prettier** rules before submission.

## 3. Commit and Push Policy

- Do **not** commit or push any code changes without explicit approval.
- Any unauthorized commits or pushes will be subject to **review**, **rollback**, or **disciplinary action**.
- Before committing or pushing, ensure that all **linting** and **TypeScript strict-mode** issues are fully resolved.
- Do **not bypass** pre-commit or pre-push checks under any circumstance.

## 4. Production Builds

- Do **not** execute a production build under any circumstances unless direct authorization has been given.

## 5. Testing and Validation

- All updates, fixes, and new features must be thoroughly **tested and validated** to ensure full functionality, stability, and compliance with project standards prior to approval.
