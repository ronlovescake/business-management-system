# 🚨 CRITICAL DEVELOPMENT RULES 🚨

## **RULE #1: NEVER COMMIT WITHOUT EXPLICIT PERMISSION**

### ❌ FORBIDDEN ACTIONS:

- **DO NOT** use `git commit` unless explicitly asked
- **DO NOT** make unauthorized commits
- **DO NOT** commit after making code changes automatically
- **DO NOT** assume changes should be committed

### ✅ ALLOWED ACTIONS:

- Make code changes when requested
- Use `git status` to show changes
- Use `git diff` to review changes
- **WAIT** for explicit approval before committing

### ✅ ONLY COMMIT WHEN USER SAYS:

- "commit this"
- "please commit"
- "make a commit"
- "save this to git"
- "commit these changes"
- Any other explicit commit instruction

### 📋 WORKFLOW:

1. User requests a change
2. Make the code changes
3. Show what changed (`git diff` or `git status`)
4. **STOP AND WAIT**
5. User tests the changes
6. User explicitly says "commit"
7. ONLY THEN make the commit

### Violations:

- Making multiple commits in a row without approval
- Committing before user has tested

- Creating documentation files and committing them without permission
